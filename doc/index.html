      Google Sign-In on Heroku

      Google OAuth fails on Heroku when the callback URL does not match what is registered in Google Cloud.

      1. Heroku Config Vars (Settings → Config Vars):

      2. GOOGLE_CLIENT_ID — same Web client as local dev (or a separate production client)

      3. GOOGLE_CLIENT_SECRET

      4. APP_HOST — your public app URL with https, no trailing slash, e.g. https://activity-finder-8073ba70e16c.herokuapp.com
        Do not leave APP_HOST as http://localhost:3000 on Heroku.
        If APP_HOST is omitted, the app falls back to https://<HEROKU_APP_NAME>.herokuapp.com (Heroku sets HEROKU_APP_NAME automatically).

      5. Google Cloud Console → Credentials → your OAuth 2.0 Web client → add for production:

      6. Authorized JavaScript origins: https://activity-finder-8073ba70e16c.herokuapp.com

      7. Authorized redirect URIs: https://activity-finder-8073ba70e16c.herokuapp.com/auth/google_oauth2/callback

      8. Redeploy after changing config vars (heroku restart or push a new release).

      9. If sign-in still fails, check logs: heroku logs --tail and look for redirect_uri_mismatch or OmniAuth warnings.
